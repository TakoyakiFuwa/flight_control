	如果改成单MCU结构需要调试烧录的固件，
	（具体在固件的“板级配置”部分）
	对软件部分缺乏了解，可能无法烧录固件
	
	选用BetaFlight作为固件烧录软件
	开源资料较为全面，板级配置较灵活
	支持单MCU结构，成功率高
	参考资料链接：https://oshwhub.com/lyh6767/board

->关于主控：STM32F405RGT6

->关于电源(power_supply)
	未知接口通讯协议，
	飞控板需要稳定5V输入，
	保留5V-5V-CURRENT-VOLTAGE-GND-GND接口
	保留焊盘
	(参考DS-009 PIXHAWK->P8.Analog Power,开源自动驾驶仪飞控板)

*	补充:资料表示无人机飞控供电来自电机驱动板(ESC)
	需要实际观察了解验证

*	可能需要额外焊接DCDC板子作为电池升压供电
	飞控板不保留DCDC焊盘位置
**	需要了解ESC板子的供电

	更新：使用电阻隔离了Analog Power的ADC电压/电流采用口，
	同时增加电阻直接连接到电源，避免固件检测不到电池的情况

	参考:https://docs.px4.io/main/zh/assembly/quick_start_pixhawk4 
	    “电源”部分，“遥控器”部分上部
	关于电源部分，官方接口为VCC-VCC-电流-电压-GND-GND
	但公司有的实际电源配置为电压-电流-VCC-VCC-GND-GND
	选择适配公司有的实际电源接口(仅我看到的那个)

引脚使用情况：
->传感器:
	>MPU6500(六轴):
		PA6->SDO
		PA5->SCL
		PA7->SDA
		PC2->CS
		PC3->INT
	>BMP280(气压计):
		PB7->SDI
		PB6->SCK
->存储：
	>TF卡座：(*注：TF卡通信采用SPI，下述仅用于区分引脚)
		PB3->CLK
		PB4->DAT0
		PB5->CMD
		PC1->DAT3
->电量：
	>电流：PC4
	>电压：PC5
->指示：
	>WS2812(LED_STRIP)：
		PB8
